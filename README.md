# Pricing Engine

FastAPI service for scraping e-commerce product pages, ingesting product data into PostgreSQL + pgvector, and generating price intelligence summaries.

## Project Structure

- `main.py`: FastAPI app and endpoints
- `extraction/crawler.py`: Scrapes domains and saves JSON output into `data_buffer/`
- `services/ingestion.py`: Reads `data_buffer/*.json`, embeds titles, and inserts rows into PostgreSQL
- `services/intelligence.py`: Queries similar products by vector distance and aggregates price stats
- `services/embedding.py`: SentenceTransformer-based text embedding
- `core/config.py`: Environment/config values
- `core/database.py`: PostgreSQL connection helper
- `utils/currency.py`: BRL price string normalization

## Requirements

- Python 3.10+
- PostgreSQL 14+ with `pgvector` extension enabled
- Linux/macOS shell (commands below use bash/zsh style)

## 1) Install Dependencies

```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
python -m playwright install
```

## 2) Configure Environment Variables

Create a `.env` file in the project root:

```env
DB_NAME=pricing_db
DB_USER=postgres
DB_PASS=postgres
DB_HOST=localhost

# Used by the crawler LLM schema generation
GEMINI_API_KEY=your_gemini_key_here

# Present in current config module
AI_API_KEY=optional_key
```

## 3) Prepare PostgreSQL + pgvector

Create database/table (adjust user/password/db name as needed):

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE IF NOT EXISTS product_listings (
  id BIGSERIAL PRIMARY KEY,
  domain TEXT NOT NULL,
  title TEXT NOT NULL,
  price_numeric DOUBLE PRECISION,
  url TEXT,
  seller TEXT,
  title_vector vector
);
```

Optional index for faster similarity search:

```sql
CREATE INDEX IF NOT EXISTS idx_product_listings_title_vector
ON product_listings USING ivfflat (title_vector vector_cosine_ops)
WITH (lists = 100);
```

## 4) Run the API

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Open docs at:

- http://localhost:8000/docs

## 5) Typical Workflow

1. Trigger scraping for one or more product listing pages.
2. Run ingestion to store extracted products in PostgreSQL.
3. Query pricing intelligence for a target product name.

### A) Start scraping

```bash
curl -X POST "http://localhost:8000/api/v1/pipeline/scrape" \
  -H "Content-Type: application/json" \
  -d '{
    "urls": [
      "https://example.com/category/headphones",
      "https://example2.com/search?q=headphones"
    ],
    "products": [
      "Sony WH-1000XM5",
      "Bose QuietComfort Ultra"
    ],
    "max_results": 10
  }'
```

### A2) Live scrape stream

Use the live streaming endpoint to watch progress as each domain is processed:

```bash
curl -N -X POST "http://localhost:8000/api/v1/pipeline/scrape/live" \
  -H "Content-Type: application/json" \
  -d '{
    "urls": [
      "https://example.com/category/headphones",
      "https://example2.com/search?q=headphones"
    ],
    "products": [
      "Sony WH-1000XM5",
      "Bose QuietComfort Ultra"
    ],
    "max_results": 10
  }'
```

This returns server-sent events like `Starting ...`, `Crawling ...`, and `Saved ...` while the scraper runs.

### B) Ingest scraped files into DB

```bash
curl -X POST "http://localhost:8000/api/v1/pipeline/ingest"
```

### C) Query pricing intelligence

```bash
curl -X POST "http://localhost:8000/api/v1/intelligence/pricing" \
  -H "Content-Type: application/json" \
  -d '{
    "product_name": "Sony WH-1000XM5",
    "similarity_threshold": 0.4
  }'
```

## Generated Runtime Folders

- `schemas/`: one schema file per domain
- `data_buffer/`: scraped JSON payloads before ingestion

These folders are created automatically when scraping starts.

## Troubleshooting

- If Playwright browser errors appear, run `python -m playwright install` again.
- If database connection fails, verify `.env` values and PostgreSQL host/user/password.
- If similarity search returns no results, try a higher `similarity_threshold` (for example `0.6`).
- If no files are ingested, check that `data_buffer/` contains `.json` files generated by scraping.

## Notes

- `services/embedding.py` uses `SentenceTransformer` and reads `MODEL_NAME` from `core/config.py`.
- The ingestion step currently keeps files in `data_buffer/` (deletion is commented out).
